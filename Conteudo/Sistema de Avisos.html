Sistema de Avisos

O sistema de aviso Embrace foi desenvolvido para que o usuário receba um notificações no celular e/ou e-mail quando um evento escolhido pelo usuário é executado na automação.

Os eventos monitorados podem ser nativos do sistema Embrace, como:

  • Movimento;
  • Iluminação acesa/apagada;
  • Ar condicionado ligado/desligado;
  • Áudio/Vídeo ligado/desligado;
  • Temperatura abaixo/acima de determinado valor.

Já em casos em que o evento a ser executado não é nativo do sistema, é possível programá-lo no E-Config.

Para isso, basta adicionar ao projeto o item Aviso, localizado em Módulos Lógicos.

Na página inicial do Aviso há algumas opções:

  • Ativação do aviso:
    • Borda 0 -> 1: o aviso é disparado quando a conexão feita é aberta.
    • Borda 1 -> 0: o aviso é disparado quando a conexão feita é fechada.
    • Ambos: o aviso é disparado em ambas situações.
  • Conexões:
    • Entrada: a conexão feita neste link é a saída de um dispositivo digital, ou seja, seu feedback.

Exemplo prático

Considere um sensor colocado em uma porta para que saibamos quando é aberta/fechada. A partir daí queremos criar um aviso que dispare uma mensagem todas as vezes que essa porta for aberta. Por não ser nativo do sistema, precisamos utilizar o E-Config.

De acordo com a opção escolhida em Ativação do aviso este será recebido em momentos diferentes:

  • Borda 0 -> 1: o aviso é disparado quando a porta for aberta.
  • Borda 1 -> 0: o aviso é disparado quando a porta for fechada.
  • Ambos: o aviso é disparado quando a porta for aberta e também quando for fechada.

Configuração na E-App

Com todos os eventos possíveis definidos, agora basta gerar o aviso na E-App:

No aplicativo Embrace 2, acesse o menu e toque em Avisos.

A tela a seguir contém todos os avisos já cadastrados. Toque no + localizado no canto superior direito para adicionar um aviso.

Na primeira tela o usuário vai definir o nome e a mensagem do aviso.

O nome tem função apenas de organização na lista de avisos e a mensagem será mostrada no conteúdo do aviso. O nome é de preenchimento obrigatório, já a mensagem pode ser deixada em branco, neste caso será gerada uma mensagem default de acordo com o evento escolhido.

  • Se o evento for nativo do sistema Embrace, a mensagem default é "<evento> no ambiente <nome do ambiente>".
  • Se o evento for feito no E-Config, a mensagem default é "<nome do aviso colocado no E-Config>".

A seguir, o usuário pode escolher como será avisado: celular e/ou e-mail.

  • Celular: o aviso será enviado ao dispositivo em questão.
    • Se o usuário for administrador pode escolher quem terá acesso ao aviso: apenas o próprio usuário ou todos os usuários (que podem habilitar/desabilitar quando quiserem).
    • Se o usuário não for administrador só terá opção de criar avisos privados.
  • E-mail: o aviso será enviado a um e-mail. Selecionando essa opção abrirá um campo para adicionar o e-mail que o usuário deseja receber o aviso. Este campo é preenchido automaticamente, mas o usuário pode alterá-lo.

Em seguida tem a lista de todos os eventos possíveis e logo após o usuário escolhe os ambientes que serão monitorados. O item anterior (eventos possíveis) filtrará os ambientes e só serão listados os ambientes que possuírem o item responsável pela ativação do evento.:

  • Movimento (o aviso será disparado na detecção de movimento);

  • Iluminação acesa (o aviso será disparado quando a iluminação for acesa);

  • Iluminação apagada (o aviso será disparado quando a iluminação for apagada);

  • Ar condicionado ligado (o aviso será disparado quando o ar condicionado for ligado);

  • Ar condicionado desligado (o aviso será disparado quando o ar condicionado for desligado);

  • Áudio/Vídeo ligado (o aviso será disparado quando um equipamento de áudio e vídeo for ligado);

  • Áudio/Vídeo desligado (o aviso será disparado quando um equipamento de áudio e vídeo for desligado);

  • Temperatura abaixo de (ao selecionar esta opção aparecerá um campo para adicionar a temperatura e o aviso será disparado quando a temperatura for menor que o valor adicionado);

  • Temperatura acima de (ao selecionar esta opção aparecerá um campo para adicionar a temperatura e o aviso será disparado quando a temperatura for maior que o valor adicionado).

Em seguida o usuário pode escolher o momento em que o aviso será monitorado:

  • Sempre: sempre que o evento for executado o aviso será enviado;
  • Intervalo de tempo: o aviso será monitorado apenas no intervalo determinado.

É possível atribuir os dias da semana que deseja ser avisado:

  • Semana inteira: o aviso será monitorado dentro das restrições colocadas independente do dia da semana;
  • Retirando a seleção do item Semana inteira é possível selecionar dias específicos da semana.

Em seguida o usuário pode determinar o período do dia que deseja receber o aviso.

  • Dia inteiro: o aviso será monitorado qualquer momento do dia.
  • Retirando a seleção do item Dia inteiro é possível selecionar o período do dia que o usuário deseja receber o aviso.

Por fim, o usuário pode determinar o tempo de silenciamento: após um disparo de aviso, o sistema ignorará a condição do aviso que foi disparado pelo tempo definido de silenciamento e não irá monitorar a condição em questão; passado esse tempo, o sistema volta a monitorar a condição do aviso até que o mesmo evento seja disparado novamente.

Ao tocar em Salvar o aviso será listado na tela inicial.

Informações adicionais

  • Os avisos serão recebidos em dispositivos que possuírem pelo menos um projeto gráfico da instalação baixados via Cloud:
    • E-App Automático: se o usuário for administrador ou possuir acesso remoto poderá acessar a tela de avisos da mesma forma feita no projeto Controle de Usuário.
    • E-App Customizado: se o usuário possuir algum aviso cadastrado e habilitado, este aviso será recebido no dispositivo que tiver o E-App Customizado baixado. Para acessar a tela de avisos basta fazer o movimento de pinça no projeto (o mesmo para acessar controle de usuário e sair do projeto) e selecionar o item Avisos. A partir daí o processo é o mesmo descrito acima.
  • Usuários administradores podem criar e editar avisos privados e públicos.
  • Usuários não administradores com acesso remoto podem apenas criar avisos privados.
  • Usuários não administradores e sem acesso remoto não têm acesso à lista de avisos e não os recebem.
  • Avisos públicos aparecem desabilitados para os demais usuários, sendo necessário habilitá-los se for desejado recebê-los.
  • Projetos baixados localmente não têm acesso aos avisos.
