Módulo NOT

Este módulo foi desenvolvido para que determinadas funções sejam executadas se os critérios estipulados pelo programador sejam falsos. O Módulo NOT tem a função de realizar a verificação dos links de Entrada e executar os comandos associados às suas conexões de Saída quando as Entradas são falsas.

Para realizar esta verificação, é oferecida a opção de selecionar entre os operadores condicionais OR/AND, o que faz com que o programador possa escolher entre esses critérios de forma a garantir que a Saída vinculada será executada apenas quando todas as Entradas forem falsas (condicional AND) ou que a  em questão poderá ser executada sempre que uma ou mais condições estipuladas na Entrada forem falsas (condicional OR)

A tela principal do Módulo NOT contém as Conexões:

  • Entrada: nesta conexão basta associar os critérios que serão verificados e que deverão ser falsos para que a Saída seja executada.
  • Saída: função a ser executada depois que se provarem falsas as Entradas estipuladas (uma ou todas, a depender do operador lógico selecionado, conforme descrito abaixo). É possível adicionar mais de uma função.

Além disso, também é oferecida a opção de selecionar o Tipo de operador lógico:

  • OR/AND: aqui é possível selecionar qual será a base para a verificação das condições. Como mencionado anteriormente, ao selecionar a condicional AND, o Módulo NOT fará a verificação de todos os critérios na Entrada e só realizará o acionamento vinculado à Saída uma vez que todos estes critérios sejam atendidos (falsos) ao mesmo tempo. Já na condicional OR, bastará que uma (ou mais) condições estipuladas sejam atendidas (falsas) para que o acionamento da Saída seja realizado.
