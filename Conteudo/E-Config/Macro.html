Macro

A Macro é um Módulo lógico que cumpre a função de armazenar (e executar) uma sequência de instruções, totalmente programável e customizável desde os seus acionadores até os comandos que serão realizados. O usuário poderá adicionar essa ferramenta ao seu projeto acessando a aba de Módulos Lógicos localizada na guia de Equipamentos.

É uma ferramenta de programação bastante intuitiva, sendo necessário apenas arrastar o comando e o item que deseja, envolvendo iluminação, drivers, equipamentos, entre outros.

Conexões

Em Entrada encontramos os itens:

  • Acionadores: dispositivo que irá acionar a macro;
  • OnChange: a macro será acionada a cada mudança de estado. Além disso, verifica se a macro está em execução. Pode estar associado a sensores, por exemplo;
  • Abortar: a macro terá sua execução abortada ao acionar esta opção.

Em Saída encontramos apenas o item Em execução, momento em que a macro é acionada e executada.

Tela de comandos

É o espaço que serão inseridos os comandos localizados à direita da janela. Esta tela possui alguns itens à direita:

  • Deletar comando;
  • Copiar comando;
  • Colar comando;
  • Recortar comando;
  • Subir comando na lista;
  • Descer comando na lista.

Basta selecionar o comando e o item desejado para que seja feita a alteração.

Entradas

São listadas todas as entradas de todos os ambientes do projeto. De acordo com o comando adicionado à tela de comandos, as entradas podem mudar. Para expandir os grupos de comandos basta clicar no símbolo mais (+) à esquerda de cada grupo e serão listadas as entradas que podem ser adicionadas ao comando utilizado.

Lista de comandos

Se ____ igual a ____, faça:

Este comando é conhecido como condicional por estabelecer condições para que determinados comandos aconteçam. Além disso, dá a chance de executar outro comando caso a situação não respeite as condições iniciais.

  • O primeiro campo é a entrada do comando, podendo ser uma Variável, um Módulo Dia/Noite, Sensores ou qualquer outra entrada que assuma valores reais.
  • O segundo campo é um combobox com as opções:
  • menor que
  • menor ou igual a
  • igual a
  • diferente de
  • maior que
  • maior ou igual a
  • O terceiro campo é o valor que a entrada do primeiro campo pode assumir.

Depois de definir as entradas, é preciso determinar o que deve acontecer.

  • Para isso basta arrastar os comandos escolhidos em cima de Faça para que sejam executados se respeitar as condições impostas.
  • Caso não respeitem, o programador pode atribuir outros comandos no campo Senão, faça ou simplesmente deixar em branco.
Acionar a entrada ____.

Este comando serve para executar entradas, como cenas, outras macros, funções de drivers, entre outros.

  • Basta arrastar o comando Acionar a entrada ____. ou simplesmente arrastar a entrada direto à tela de inserção de comandos.
  • Algumas entradas só podem ser acionadas através de Hold e neste comando é possível simular essa situação colocando o tempo de Hold, em segundos, no segundo campo.
Atribuir valor xxxx à entrada

Com este comando, além de controlar as entradas dos diversos equipamentos disponíveis no E-Config, também é possível controlar Variáveis.

Enquanto ____ igual a ____, faça:

Este comando é conhecido como loop, pois irá repetir os comandos adicionados enquanto  respeitarem determinadas condições.

  • O primeiro campo é a entrada do comando, podendo ser uma Variável, um Módulo Dia/Noite, Sensores ou qualquer outra entrada que assuma valores reais.
  • O segundo campo é um combobox com as opções:
  • menor que
  • menor ou igual a
  • igual a
  • diferente de
  • maior que
  • maior ou igual a
  • O terceiro campo é o valor que a entrada do primeiro campo pode assumir.
Repetir os comandos por xxxx vezes.

Como o próprio nome diz, este comando serve para repetir os comandos. Podem ser adicionados quaisquer comandos para serem repetidos a quantidade de vezes determinada.

Aguarde por xxxx segundos.

Com este comando é possível adicionar tempo entre quaisquer comandos.

Jump para xxxx, xxxx

Este comando serve para redirecionar para um Guia TV ou um aplicativo externo.

  • O primeiro campo é um combobox com as opções GuiaTV e aplicativo externo. O segundo campo depende da escolha do primeiro.
  • Guia TV
    • Com essa opção, o segundo campo listará os drivers que foram adicionados ao projeto que têm Guia TV.
    • Ao acionar esta macro pelo aplicativo Embrace abrirá uma página com os canais e a programação do Guia.
  • Aplicativo externo
    • O segundo campo listará os aplicativos testados pela Scenario.
    • Nem todos estão disponíveis para ambos os sistemas (iOS e Android).
