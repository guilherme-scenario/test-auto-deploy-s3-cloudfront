Drivers

A plataforma Embrace oferece a possibilidade de integração dos mais diversos tipos de dispositivos à automação por meio de drivers: no projeto Embrace, um driver atua como uma interface de comunicação entre o dispositivo e a automação por meio de diferentes vias de comunicação, como IP, IR e RS232 (Serial).

Além dos dispositivos da própria linha Embrace, a plataforma permite a integração com dispositivos de outros fabricantes por meio destes drivers, o que faz com que o projeto de automação possa controlar uma vasta quantidade de aparelhos dos mais variados tipos, como dispositivos de áudio e vídeo e ar-condicionado, por exemplo.

Projeto

Para realizar o controle de dispositivos por meio da automação, basta adicionar um driver compatível com o dispositivo que deseja controlar ao projeto Embrace (via E-Config)e configurá-lo de acordo com o método de comunicação que o aparelho usa (selecionando a Rede para o caso de dispositivos que comunicam via IR/Serial ou preenchendo o Endereço IP para dispositivos que comunicam via IP, por exemplo).

Biblioteca Scenario

A Scenario constantemente atualiza sua biblioteca de drivers visando expandir a gama de dispositivos suportados e garantir cada vez mais recursos ao usuário.

Biblioteca Usuário

Da mesma forma, o Usuário também consegue gerenciar sua própria biblioteca de drivers, criando-os ou personalizando drivers já existentes para melhor atender sua instalação.

Os drivers Scenario possuem templates próprios que são exibidos no projeto gráfico da E-App. Da mesma forma que ocorre com outros dispositivos de projeto Embrace, os drivers também possuem Conexões de Entrada e Saída para que seus acionamentos possam ser programados de acordo com a necessidade do projeto de automação.

Gerenciamento de Drivers

No menu de Gerenciamento de Drivers do E-Config, o usuário pode Criar, editar, importar e exportar drivers, bibliotecas e templates.

Além de garantir esses recursos, o menu de Gerenciamento de Drivers também permite ao usuário ter uma visão mais completa dos drivers que compõem as bibliotecas às quais ele tem acesso, com detalhes específicos pertinentes a cada comando desses drivers a nível individual.

Da mesma forma, o E-Config também oferece a possibilidade de testar esses comandos sem que seja necessário testar o driver diretamente por meio do projeto, auxiliando o usuário no processo de desenvolvimento e/ou ajuste de drivers.

Para realizar um teste de driver através do menu de Gerenciamento, basta selecionar o driver desejado (seja ele oriundo da biblioteca Scenario ou da própria biblioteca do Usuário) e selecionar a opção Editar e logo em seguida a opção Iniciar Teste.

Após Iniciar o teste, algumas novas opções serão exibidas na tela de Comandos do driver em questão:

  • A caixa de seleção de Redes lista todos os dispositivos emissores da rede que estão disponíveis para realizar o teste;
  • A caixa de seleção de Canal lista os canais disponíveis que emitirão os comandos de teste (Exemplo: Canais 1-8 da EB-NTL1 ou IR Blaster/Externo da EB-IRS-WIFI);
  • O ícone de varredura realizar uma nova busca na rede da Controladora para atualizar a lista de Redes disponíveis;
  • A coluna Enviar possui ícones ao lado de cada comando do driver. Ao clicar no ícone, o comando em questão é disparado por meio do Canal da Rede selecionada.

Templates

Um Template é um agrupamento de componentes gráficos (Ex: botões, sliders e texto) que são exibidos no projeto da E-App com o propósito de agir como a interface dos drivers no aplicativo: dessa forma, é possível interagir com os drivers da automação e realizar seus acionamentos desejados via E-App.

A Scenario oferece uma biblioteca de templates próprios tanto para E-App quanto para o EB-RC1-Remote a fim de garantir ao usuário a possibilidade de Criar novos drivers ou editar drivers existentes utilizando os modelos genéricos desta biblioteca.

Por meio do Gerenciador de Templates, é possível Criar, editar, importar e exportar templates tanto para E-App quanto para o EB-RC1-Remote.

Quando um usuário tenta Criar ou Editarum template, o E-Config automaticamente executa o Editor Gráfico, que é a ferramenta que permite realizar os ajustes necessários nos componentes gráficos que pertencem ao template. Esses componentes eventualmente devem ser associados aos Comandos dos drivers (ao Visualizar o template e arrastar os componentes gráficos até os comandos desejados) para que o projeto gráfico na E-App interaja com o driver.

Além do ajuste automático, o E-Config também oferece recursos para a personalização de templates tanto para E-App quanto para o Remote. Com esses recursos, é possível alterar o esquema de organização de colunas e paginação dos templates: para personalizar esses elementos, basta utilizar etiquetas de Texto com palavras reservadas que vão estabelecer parâmetros que serão considerados quando o ajuste automático for realizado no template. Quando uma palavra reservada é aplicada no template, todos os elementos gráficos que estão diretamente abaixo do componente de Texto serão afetados. Vale enfatizar que esses recursos de formatação são inteiramente optativos, portanto seu uso não é obrigatório para que o template que está sendo adaptado seja considerado compatível.

Abaixo estão as definições de cada palavra reservada:

  • ##3 colunas##

    Ao inserir no template um componente de Texto com este conteúdo, todos os elementos gráficos posicionados diretamente abaixo deste componente serão automaticamente ordenados pelo Editor Gráfico de forma que a grade de visualização vertical do template fique igualmente dividida em três partes. Esta é a visualização default dos templates para o Remote, portanto as palavras reservadas com parâmetros de colunas (como essa) não são aplicáveis nestes tipos de template.

  • ##4 colunas##

    Ao inserir no template um componente de Texto com este conteúdo, todos os elementos gráficos posicionados diretamente abaixo deste componente serão automaticamente ordenados pelo Editor Gráfico de forma que a grade de visualização vertical do template fique igualmente dividida em quatro partes.

  • ##muda pagina##

    Ao inserir no template um componente de Texto com este conteúdo, todos os elementos gráficos posicionados diretamente abaixo deste componente serão exibidos em uma nova página, navegável via E-App e Remote por meio do gesto de swipe lateral.

  • ##pagina emergencia##

    Ao inserir no template um componente de Texto com este conteúdo, todos os elementos gráficos posicionados diretamente abaixo deste componente serão automaticamente ordenados pelo Editor Gráfico de forma que eles passem a ser exibidos na Página de Emergência do driver (mais informações na seção dedicada abaixo).

Página de Emergência

O recurso de Página de Emergência é oferecido com base no conceito de que o acesso a algumas funções avançadas no controle de dispositivos por meio da automação não é interessante a todo momento e para todos os usuários (por motivos como sincronia de feedback e preservação de configurações, dentre outros), porém ainda assim é pertinente em casos pontuais, e, portanto, também devem estar presentes em uma página dedicada do template do driver para eventual acesso, à qual atribui-se o nome de Página de Emergência.

Nesta página geralmente ficarão localizadas as funções que podem afetar a sincronia do dispositivo com a automação e, portanto, possui a opção de ser ocultada ou não.

Para alterar essa opção, basta marcar ou desmarcar a checkbox pertinente na tela de Programação do driver. Uma vez habilitada, a Página de Emergência (bem como as demais páginas convencionais pertencentes a um determinado template) pode ser acessada por meio do gesto de swipe lateral.
